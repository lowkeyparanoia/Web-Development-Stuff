Position Scheme

This website use a mixture of positioning to better fit the content.

Most of the elements use static positioning.

Below is an explanation of other positioning methods used:

Absolute positioning
    the form input and select in home page is positioned with 10% right empty space (relative to the form) to be aligned vertically. This also takes the elements out of normal flow. (homepage.css, Line 98)
    fixed positioning
    the nav class and the navigation icon is fixed to the top right corner of the browser body. This also takes the elements out of normal flow. (navigation.css, Line 42)
Relative positioning
  • the navigation bar, a child of nav element, is positioned lower, relative to its original place (relative positioning), to ensure it is not covered by the navigation icon. (z-index) (navigation.css, Line 2 on VSCode)
  • the form in home page is set to relative to serve as positioning ancestor for the input and select. (see the next item, homepage.css, Line 83 on VSCode)

Style grouping

All the CSS files are in 6 files in the /css folder, and HTML files are linked to them.

No inline/embedded styles is used in this website. It isn't a recommended practice.

The files are as follows:

default.css
The common theme across the site and other links (keeps design consistent, sets sizing precedence), for index.html and order.html (on screen).
Including the header and welcome (Line 21-32, 48-52), footer (Line 31-38) and border style (Line 2-12).
OrderPrint.css
Styling applied for printing the order.html.
All the blocks other than the header and ordered item are not hidden/not displayed.
These two blocks are also auto adjusted to fit the print width. Header (Line 3-25), ordered item (Line 32-62).
homepage.css
Specific style for index.html.
Including the two-column layout (Line 4-12), information block (Line 15-63), promotion block with the iframe as a service for location (Line 66-75), booking block (Line 78-115).
order.css
Specific style for order.html (on screen).
including the two-column layout (Line 4-12), order menu (Line 19-94) and ordered item (Line 97-127).
navigation.css
Navigation styles for the nav menu with linked pages, for all pages (on screen, that is, will be hidden in print).
Including the navigation icon (Line 3-15), navigation menu (Line 16-47) and keyframe functions (Line 49-70).
design.css
Specific styling for this page, design.html.

CSS3 Features

Rounded Corner Borders
  • border-radius is used to create rounded corners for many div and box elements. (default.css, Line 3-18, with the border selectors)
  • border-radius is also used for labelling the menu in the order page, with only topleft and topright corner rounded. (order.css, Line 36)
Transitions
  • transition is used to create a color and background color change effect when the mouse hovers over an item in the navigation menu. (navigation.css, Line 91-94)
  • A similar, background color change transition is applied for the menu label in order page. (order.css, Line 70)
Animations
  • When the page is loaded, animation makes the navigation icon flys in with rotation, and then rotate for another 360 degrees. (navigation.css, Line 62-80)
  • At the same time, the navigation menu appears forwards with a cubic-bezier function. (navigation.css, Line 25-30)
